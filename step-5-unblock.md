OBJECTIVE

Unblock Step 5 (Data Ingestion) by safely injecting a valid OpenAI API key at runtime on the Weaviate EC2 instance, without persisting the key in any file, Terraform config, or repository.

This is a controlled, ephemeral secret injection for one-time ingestion only.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
CRITICAL SECURITY RULES (NON-NEGOTIABLE)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- DO NOT print, log, or echo the OpenAI API key at any time.
- DO NOT write the key to disk (.env, .bashrc, files, or scripts).
- DO NOT modify terraform.tfvars.
- DO NOT modify any repository files.
- DO NOT include the key in any command history or output.
- The key must exist ONLY as an in-memory environment variable in a single shell session.
- Immediately remove the key after ingestion completes.

If any step would violate these rules, STOP and report.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PREREQUISITES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- A valid OpenAI API key has been generated by the user.
- SSH access to the Weaviate EC2 instance is confirmed.
- Weaviate is running and reachable at http://localhost:8080.
- faq_schema.json is already applied (Step 4 complete).

If any prerequisite is not met, STOP and report.

STEP 1 — SSH INTO WEAVIATE EC2
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Establish SSH session:

ssh -i ~/.ssh/<key_name>.pem <user>@<WEAVIATE_EC2_IP>

If SSH fails, STOP and report.

STEP 2 — RUNTIME SECRET INJECTION (IN-MEMORY ONLY)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

In the active SSH session:

- Prompt the user to manually paste the OpenAI API key into the terminal.
- DO NOT capture or display the key in any output.

Set environment variable:

export OPENAI_APIKEY='<PASTE_KEY_HERE>'

Immediately verify variable is set WITHOUT revealing value:

test -n "$OPENAI_APIKEY" && echo "OPENAI_APIKEY set" || echo "OPENAI_APIKEY missing"

If missing, STOP and report.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 3 — VERIFY INGESTION PREREQUISITES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Before running ingestion:

1. Confirm repository path:

cd <repository_root>

2. Confirm ingestion script exists:

ls ingest-faqs-batched.py

3. Confirm dataset availability:

- If script expects local file:
  ls faqs.csv

- If pulled from S3:
  aws s3 ls s3://weaviate-faq-csv-v24

If any required file is missing, STOP and report.

4. Confirm Weaviate is reachable:

curl -s http://localhost:8080/v1/meta | python3 -m json.tool

Expected: valid JSON with "version"

If not reachable, STOP and report.
